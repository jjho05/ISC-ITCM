Ingeniería de Software

Clave: SCD-1011 | Créditos SATCA: 2-3-5

Caracterización de la Asignatura

Esta asignatura aporta al perfil del Ingeniero en Sistemas Computacionales habilidades para implementar aplicaciones, diseñar interfaces, coordinar equipos multidisciplinarios y administrar software para apoyar la productividad de las organizaciones cumpliendo con estándares de calidad.

Su importancia radica en que permite al estudiante aplicar las fases de la metodología para el desarrollo de un sistema en un contexto multidisciplinario, aplicando el conocimiento científico, a través de los métodos, técnicas y normas adecuados. Se relaciona con materias como Fundamentos de Programación, Estructura de Datos y Taller de Bases de Datos, y sienta las bases para Gestión de Proyectos de Software y Programación Web.

Competencia General de la Asignatura

  • engineering Desarrolla soluciones de software, considerando la metodología y herramientas para la elaboración de un proyecto aplicativo en diferentes escenarios.

Temario y Competencias por Unidad

1. Análisis

Competencia: Abstrae información del usuario final para elaborar el análisis de requerimientos del software a desarrollar.

  • 1.1 Revisión de especificación de requisitos.
    • 1.1.1 Norma IEEE830
    • 1.1.2 Trazabilidad de requisitos
  • 1.2 Descripción de procesos actuales
  • 1.3 Diagramas UML
  • 1.4 Estudio de Factibilidad
  • 1.5 Análisis Costo-Beneficio

2. Diseño

Competencia: Aplica modelos, técnicas y herramientas para la etapa de diseño del software.

  • 2.1 Diseño de procesos propuestos
    • 2.1.1 Herramientas CASE para diseño
  • 2.2 Diseño arquitectónico
  • 2.3 Diseño de datos
  • 2.4 Diseño de interfaz de usuario

3. Desarrollo

Competencia: Construye un software derivado de la problemática planteada en el análisis y diseño de un sistema.

  • 3.1 Lenguajes para el desarrollo de software
  • 3.2 Manejadores de bases de datos
  • 3.3 Construcción del sistema

4. Pruebas e Implantación

Competencia: Evalúa el software construido a partir de las pruebas e implementación realizada.

  • 4.1 Diseño de caso de prueba
  • 4.2 Pruebas de componentes
  • 4.3 Pruebas del sistema
  • 4.4 Documentación de resultados de las pruebas.
  • 4.5 Entrega del sistema y Capacitación a usuarios
  • 4.6 Entrega de documentación técnica y de usuario del sistema.

Fuentes de Información Sugeridas

Textos recomendados para esta asignatura:

  • Pressman, R.S. (2008). Ingeniería del Software un enfoque práctico. 6ª. Edición. MC GRAW HILL.
  • Sommerville, Ian (2001). Ingeniería de Software. PRENTICE HALL.
  • Booch G. (2006). El lenguaje Unificado de Modelado, UML 2.0, Guía de Usuario. 2ª. Edición. Pearson ADDISON-WESLEY.
  • Kendall E. K. (2011). Análisis y Diseño de sistemas. 8ª. Edición. PEARSON.
  • Senn J.A. (1996). Análisis y Diseño de Sistemas. 2ª Edición. MC GRAW HILL.
