Taller de Sistemas Operativos

Clave: SCA-1026 | Créditos SATCA: 0-4-4

Caracterización de la Asignatura

Esta asignatura aporta al perfil del egresado habilidades para implementar aplicaciones computacionales y para diseñar, configurar y administrar redes, creando soluciones de conectividad. El estudiante obtendrá los conocimientos para la administración de diferentes sistemas operativos, con el propósito de brindar alternativas de solución a problemas reales en la industria.

La aportación de la materia se basa en emplear competencias previas de la asignatura de sistemas operativos, para que el estudiante posea un criterio base para la elección del sistema operativo a emplear. Las competencias desarrolladas le permitirán instalar y administrar sistemas operativos para la implementación futura de servicios de red y su monitorización.

Competencia General de la Asignatura

  • dns Conoce, identifica, selecciona y administra diferentes sistemas operativos para resolver problemáticas reales, y aplica procedimientos de interoperabilidad entre ellos.

Temario y Competencias por Unidad

1. Introducción a los sistemas operativos

Competencia: Identifica los diferentes tipos de sistemas operativos que se emplean en la industria para su aplicación según la problemática a resolver.

  • 1.1 Clasificación y Estructuras genéricas de los Sistemas Operativos vigentes.
  • 1.2 Procesos y Multiprogramación.
  • 1.3 Virtualización.
    • 1.3.1 Componentes y Niveles de Virtualización.
    • 1.3.2 VPS (Virtual Private Server).

2. Sistemas Operativos propietarios para servidores

Competencia: Instala, configura y administra un sistema operativo propietario para resolver una necesidad determinada, considerando la planeación de mantenimientos y recuperaciones.

  • 2.1 Características y Análisis de los Sistemas Operativos Propietarios.
  • 2.2 Requerimientos de instalación.
  • 2.3 Configuración Básica.
  • 2.4 Comandos Básicos y aplicaciones.
  • 2.5 Administración del Sistema.
    • 2.5.1 Administración de procesos, red, memoria, sistemas de archivos, servicios, usuarios, grupos y permisos.
  • 2.6 Medición y Desempeño del Sistema Operativo.
  • 2.7 Seguridad e Integridad.
  • 2.8 Normatividad y Políticas de uso.

3. Sistemas Operativos de software libre para servidores

Competencia: Instala, configura y administra un sistema operativo de software libre para resolver una necesidad determinada, considerando la planeación de mantenimientos y recuperaciones.

  • 3.1 Características y Análisis de los Sistemas Operativos de software libre.
  • 3.2 Requerimientos de instalación.
  • 3.3 Configuración Básica.
  • 3.4 Comandos Básicos y aplicaciones.
  • 3.5 Administración del Sistema.
    • 3.5.1 Administración de procesos, red, memoria, sistemas de archivos, servicios, usuarios, grupos y permisos.
  • 3.6 Medición y Desempeño del Sistema Operativo.
  • 3.7 Seguridad e Integridad.
  • 3.8 Normatividad y Políticas de uso.

4. Interoperabilidad entre sistemas operativos

Competencia: Identifica y aplica diferentes mecanismos de interoperabilidad y exposición de recursos.

  • 4.1 Interoperabilidad entre sistemas operativos.
    • 4.1.1 Sistemas de Archivos y Recursos (NFS, Impresoras).
    • 4.1.2 Comunicación entre procesos (Sockets, RPC).
