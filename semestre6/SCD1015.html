Lenguajes y Autómatas I

Clave: SCD-1015 | Créditos SATCA: 2-3-5

Caracterización de la Asignatura

Esta asignatura aborda los temas relacionados con la teoría de lenguajes formales, lo que permite vislumbrar los procesos inherentes y, a veces, escondidos dentro de todo lenguaje. Esta materia sirve de preámbulo para la asignatura de Lenguajes y Autómatas II, donde se completa el estudio formal de la teoría de lenguajes. Las asignaturas directamente vinculadas son Estructura de Datos y Arquitectura de Computadoras, que dotan al estudiante de las herramientas y conocimientos necesarios para el desarrollo de las prácticas.

Competencia General de la Asignatura

  • hub Define, diseña y programa las fases del analizador léxico y sintáctico de un traductor o compilador para preámbulo de la construcción de un compilador.

Temario y Competencias por Unidad

1. Introducción a la Teoría de Lenguajes Formales

Competencia: Identifica los conceptos de lenguajes formales para comprender las fases de un compilador y traductor.

  • 1.1 Alfabeto.
  • 1.2 Cadenas.
  • 1.3 Lenguajes, tipos y herramientas.
  • 1.4 Estructura de un traductor
  • 1.5 Fases de un compilador

2. Expresiones Regulares

Competencia: Crea y reconoce Expresiones Regulares para solucionar problemas del entorno.

  • 2.1. Definición formal de una ER.
  • 2.2. Diseño de ER.
  • 2.3. Aplicaciones en problemas reales.

3. Autómatas Finitos

Competencia: Crea y reconoce autómatas finitos en un lenguaje de programación para la solución de un problema.

  • 3.1 Conceptos: Definición y Clasificación de Autómata Finito (AF).
  • 3.2 Conversión de un Autómata Finito No Determinista (AFND) a Autómata Finito Determinista (AFD).
  • 3.3 Representación de ER usando AFND
  • 3.4 Minimización de estados en un AF
  • 3.5 Aplicaciones (definición de un caso de estudio).

4. Análisis Léxico

Competencia: Construye un analizador léxico a partir de un lenguaje de programación.

  • 4.1 Funciones del analizador léxico.
  • 4.2 Componentes léxicos, patrones y lexemas.
  • 4.3 Creación de Tabla de tokens.
  • 4.4 Errores léxicos.
  • 4.5 Generadores de analizadores Léxicos.
  • 4.6 Aplicaciones (Caso de estudio).

5. Análisis Sintáctico

Competencia: Construye un analizador sintáctico a partir de un lenguaje de programación.

  • 5.1 Definición y clasificación de gramáticas.
  • 5.2 Gramáticas Libres de Contexto (GLC).
  • 5.3 Árboles de derivación.
  • 5.4 Formas normales de Chomsky.
  • 5.5 Diagramas de sintaxis
  • 5.6 Eliminación de la ambigüedad.
  • 5.7 Tipos de analizadores sintácticos
  • 5.8 Generación de matriz predictiva (cálculo first y follow)
  • 5.9 Manejo de errores
  • 5.10 Generadores de analizadores sintácticos

6. Máquinas de Turing

Competencia: Diseña y construye o simula una Maquina de Turing (MT), para el reconocimiento de cadenas propias de lenguajes.

  • 6.1 Definición formal MT
  • 6.2 Construcción modular de una MT
  • 6.3 Lenguajes aceptados por la MT.

Software y Herramientas Propuestos

Para las prácticas de analizadores léxicos y sintácticos, se recomienda el uso de lenguajes de programación y herramientas generadoras.

  • Lenguajes de programación de propósito general (Java, C++, etc.).
  • Generadores de analizadores léxicos y sintácticos como JFlex, CUP, ANTLR, Flex, Bison.
  • Software de simulación para autómatas como JFLAP.
