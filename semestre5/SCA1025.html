Taller de Base de Datos

Clave: SCA-1025 | Créditos SATCA: 0-4-4

Caracterización de la Asignatura

Esta asignatura aporta al perfil del Ingeniero en Sistemas Computacionales las competencias para implementar bases de datos y apoyar la toma de decisiones conforme a las normas vigentes de manejo y seguridad de la información. Consiste en el uso de lenguajes de definición, manipulación, control de acceso, transacciones y SQL procedural. Proporciona las bases para otras asignaturas vinculadas con el desarrollo de software, aplicando los temas en la definición de esquemas de bases de datos relacionales y la manipulación de información en ambientes transaccionales.

Competencia General de la Asignatura

  • hub Implementa bases de datos para apoyar la toma de decisiones considerando las reglas de negocio.

Temario y Competencias por Unidad

1. Lenguaje de definición de datos

Competencia: Utiliza procedimientos de instalación de SGBD y construye esquemas de base de datos para proyectar las necesidades de un cliente.

  • 1.1 Instalación de un SGBD
  • 1.2 Creación y modificación del esquema de la base de datos y aplicación de constraints.

2. Lenguaje de manipulación de datos

Competencia: Construye expresiones en SQL para resolver necesidades de recuperación de información con las reglas sintácticas del lenguaje.

  • 2.1 Inserción, eliminación y modificación de registros
  • 2.2 Consultas
  • 2.3 Funciones, conversión, agrupamiento, ordenamiento
  • 2.4 Joins
  • 2.5 Subconsultas
  • 2.6 Operadores set
  • 2.7 Vistas

3. Control de acceso

Competencia: Implementa mecanismos de seguridad básicos para el acceso a datos mediante el otorgamiento o denegación de privilegios.

  • 3.1 Tipos de usuario
  • 3.2 Creación de usuarios
  • 3.3 Privilegios a usuarios
  • 3.4 Roles

4. Concurrencia

Competencia: Controla la concurrencia de la base de datos, para disminuir los problemas de desempeño y/o consistencia.

  • 4.1 Conceptos
  • 4.2 Propiedades de las transacciones
  • 4.3 Grados de consistencia
  • 4.4 Niveles de aislamiento
  • 4.5 Commit y rollback

5. SQL Procedural

Competencia: Aplica SQL procedural para automatizar reglas de negocio y garantizar la integridad, consistencia y seguridad de los datos.

  • 5.1 Stored Procedures
  • 5.2 Functions
  • 5.3 Triggers

6. Conectividad de Bases de Datos

Competencia: Establece conexiones entre el SGBD y algún lenguaje de programación, mediante cadenas de conexión y/o protocolos de comunicación.

  • 6.1 ODBC, ADO.NET, JDBC
  • 6.2 Conectividad desde un lenguaje huésped o en dispositivos móviles

Software y Herramientas Propuestos

Se requiere la instalación y uso de un Sistema Gestor de Bases de Datos (SGBD) y una herramienta de administración para realizar las prácticas del curso.
