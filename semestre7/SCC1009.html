Gestión de Proyectos de Software

Clave: SCG-1009 | Créditos SATCA: 3-3-6

Caracterización de la Asignatura

Esta asignatura proporciona al estudiante los conceptos que requiere para la gestión de un proyecto de software y le da la posibilidad de poner en práctica dicha gestión. Se sugiere que el estudiante desarrolle un proyecto de gestión para una organización, adquiriendo las competencias necesarias para estar al frente de dichos proyectos.

La intención es que los estudiantes gestionen un proyecto de software de carácter multidisciplinario para trabajar las competencias genéricas de su formación profesional. Se relaciona con asignaturas previas como Taller de Administración e Ingeniería de Software, y sirve de base para asignaturas posteriores.

Competencia General de la Asignatura

  • fact_check Aplica metodologías e instrumentos para garantizar la gestión adecuada de un proyecto de software.

Temario y Competencias por Unidad

1. Introducción a la gestión de proyectos

Competencia: Conoce y comprende el entorno de la gestión de proyectos.

  • 1.1 Conceptos básicos para la gestión de proyectos.
  • 1.2 Fases de la gestión de proyectos.
  • 1.3 Fundamentos de Project Management Institute.

2. Gestión de calidad

Competencia: Identifica y selecciona estándares y métricas de calidad para ser aplicados a un proyecto de software.

  • 2.1 Plan de calidad del software.
  • 2.2 La gestión de proyectos usando un marco de calidad.
  • 2.3 Estándares y Métricas de calidad (CMMI, MoProSoft).
  • 2.4 Impacto de la calidad en tiempo, costo y alcance.
  • 2.5 Control del cambio.

3. Planificación del proyecto

Competencia: Planifica un proyecto de software utilizando una metodología de trabajo para determinar su viabilidad.

  • 3.1 Objetivo del proyecto.
  • 3.2 Estimaciones de tiempo, costos y personal.
  • 3.3 Análisis de riesgos.
  • 3.4 Análisis de la viabilidad del proyecto.

4. Presentación de la información

Competencia: Identifica y selecciona propuestas de proyecto de software para la presentación de un contrato al cliente.

  • 4.1 Propuesta.
  • 4.2 Lineamientos de comunicación y seguimiento.
  • 4.3 Contrato.

5. Supervisión y Revisión del proyecto

Competencia: Aplica políticas de comunicación y seguimiento para la mejora del proyecto de software.

  • 5.1 Administración de actividades.
  • 5.2 Administración del tiempo.
  • 5.3 Evaluación y ajustes del proyecto.

Software y Herramientas Propuestos

Para la gestión y planificación de proyectos de software, se recomienda el uso de herramientas especializadas que facilitan el seguimiento y la colaboración.

  • Software de gestión de proyectos como Microsoft Project, Jira, Trello, o Asana.
  • Herramientas CASE para el modelado y la documentación del proyecto.
  • Sistemas de control de versiones como Git para la gestión de artefactos del software.
  • Software de ofimática para la elaboración de reportes, propuestas y presentaciones.
